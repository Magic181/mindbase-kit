# 数据库设计

> 最后更新：2026-06-27

当前数据库模型位于 `backend/apps/*/models.py`，支持 MySQL 和 SQLite。生产推荐 MySQL，本地快速体验可使用 SQLite。

## ER 概览

```
User
  └── Notebook
        ├── Document
        │     ├── DocumentChunk
        │     └── DocumentAsset
        └── Conversation
              └── Message
```

## notebooks_notebook

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 所属用户 |
| name | VARCHAR(100) | Notebook 名称 |
| description | TEXT | 描述 |
| is_favorite | BOOLEAN | 是否收藏 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

索引：

- `user, is_favorite`
- `user, updated_at`

## documents_document

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| notebook_id | BIGINT | 所属 Notebook |
| name | VARCHAR(255) | 原始文件名 |
| file_path | VARCHAR(500) | 本地存储相对路径 |
| file_size | BIGINT | 文件大小 |
| file_type | VARCHAR(20) | `txt` / `md` / `pdf` / `docx` |
| status | VARCHAR(20) | `uploading` / `parsing` / `completed` / `failed` |
| chunk_count | INT | 当前分块数量 |
| error_message | TEXT | 文档级解析错误 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## documents_documentchunk

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| document_id | BIGINT | 所属文档 |
| content | TEXT | 检索文本 |
| metadata | JSON | source_type、页码、表格、代码、图片等元数据 |
| position | INT | 文档内位置 |

常见 `metadata.source_type`：

- `paragraph`
- `heading`
- `table`
- `code`
- `page`
- `image_ocr`
- `image_caption`

## documents_documentasset

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| document_id | BIGINT | 所属文档 |
| asset_type | VARCHAR(20) | 当前为 `image` |
| file_path | VARCHAR(500) | 图片资源路径 |
| original_name | VARCHAR(255) | 图片原名或生成名 |
| position | INT | 文档内资源位置 |
| nearby_text | TEXT | 图片附近文本 |
| ocr_status | VARCHAR(20) | `pending` / `success` / `skipped` / `failed` |
| ocr_text | TEXT | OCR 结果 |
| ocr_error | TEXT | OCR 错误或跳过原因 |
| vision_status | VARCHAR(20) | `pending` / `success` / `skipped` / `failed` |
| vision_text | TEXT | 视觉描述 |
| vision_error | TEXT | 视觉错误或跳过原因 |
| metadata | JSON | 页码、alt text、来源等 |
| created_at | DATETIME | 创建时间 |

用途：

- 为文档卡片提供 OCR/视觉处理诊断。
- 为 RAG 生成 `image_ocr` 和 `image_caption` 片段。
- 支持旧文档重新解析后刷新图片理解结果。

## chat_conversation

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| notebook_id | BIGINT | 所属 Notebook |
| title | VARCHAR(100) | 会话标题 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## chat_message

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| conversation_id | BIGINT | 所属会话 |
| role | VARCHAR(20) | `user` / `assistant` |
| content | TEXT | 消息内容 |
| citations | JSON | 文档或网页引用来源 |
| created_at | DATETIME | 创建时间 |

`citations` 中的文档来源会包含检索诊断，例如 `retrieval_score`、`retrieval_reason`、`source_type`、`vision_model`。

## 设计边界

- 当前没有独立 Tag 表；收藏通过 `Notebook.is_favorite` 实现。
- 当前没有向量表或向量 ID；向量数据库属于后续升级。
- 上传文件和解析出的图片存储在本地 `backend/media/`，没有对象存储依赖。
