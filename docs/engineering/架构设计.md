## 模块职责

### Frontend (Vue3)

- 用户界面渲染
- 状态管理 (Pinia)
- 路由管理 (Vue Router)
- API调用封装

### Backend (Django)

- RESTful API
- 用户认证授权
- 业务逻辑处理
- 异步任务调度 (Celery)

### AI Module

- RAG 检索增强生成
- 文档解析与文本分块
- DeepSeek Chat API 集成
- Tavily 联网搜索
- Prompt 模板管理

### Storage

- MySQL / SQLite: 用户、Notebook、文档元数据
- 本地 media 目录: 开发环境文件存储
- Redis: Celery broker / result backend
- Milvus / 向量索引: 后续规划

## 数据流

### 文档上传流程

```
用户上传 → Backend接收 → 本地 media 存储 → 创建文档记录
    → 触发 Celery 任务 → 文档解析 → 文本分块
    → 更新文档状态
```

### AI问答流程

```
用户提问 → Backend接收 → Notebook 文档关键词检索
    → 可选 Tavily 联网搜索 → 构建 Context → 调用 DeepSeek
    → 保存消息 → 返回文档 / 网页引用来源
```

## 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | Vue3 + Vite | 响应式、生态完善 |
| UI库 | Element Plus | 二次设计、定制化 |
| 样式 | Tailwind CSS | 原子化、高效 |
| 桌面版 | Electron | 跨平台打包，复用Web代码 |
| 后端 | Django + DRF | 成熟稳定、REST支持 |
| 任务队列 | Celery + Redis | 异步处理、可靠 |
| 关系数据库 | MySQL | 稳定、生态 |
| 向量数据库 | 暂未接入 | 后续升级为 Milvus / ChromaDB |
| 文件存储 | Django media | 本地开发简单可靠 |

### 认证机制

- JWT（SimpleJWT）+ Token 黑名单
- 注册自动登录，返回 access + refresh
- 前端主动刷新（剩余 < 60s）+ 401 重试
- 应用启动时 `/auth/me/` 校验会话
