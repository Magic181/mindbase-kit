## 模块职责

### Frontend (Vue3)

- 用户界面渲染
- 状态管理 (Pinia)
- 路由管理 (Vue Router)
- API调用封装

### Backend (Django)

- RESTful API
- 用户认证授权
- 业务逻辑处理
- 异步任务调度 (Celery)

### AI Module

- RAG检索增强生成
- 文档解析与向量化
- LLM多模型集成
- Prompt模板管理

### Storage

- MySQL: 用户、Notebook、文档元数据
- Milvus/ChromaDB: 文档向量索引
- MinIO: 文件对象存储
- Redis: 缓存、会话、消息队列

## 数据流

### 文档上传流程

```
用户上传 → Backend接收 → 存储到MinIO → 创建文档记录
    → 触发Celery任务 → 文档解析 → 文本分块 → 向量化
    → 存入Milvus → 更新文档状态
```

### AI问答流程

```
用户提问 → Backend接收 → 向量化问题 → Milvus检索
    → 获取相关文档块 → 构建Context → 调用LLM
    → 流式返回 → 标注引用来源
```

## 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | Vue3 + Vite | 响应式、生态完善 |
| UI库 | Element Plus | 二次设计、定制化 |
| 样式 | Tailwind CSS | 原子化、高效 |
| 桌面版 | Electron | 跨平台打包，复用Web代码 |
| 后端 | Django + DRF | 成熟稳定、REST支持 |
| 任务队列 | Celery + Redis | 异步处理、可靠 |
| 关系数据库 | MySQL | 稳定、生态 |
| 向量数据库 | Milvus | 生产级、可扩展 |
| 对象存储 | MinIO | S3兼容、私有部署 |
