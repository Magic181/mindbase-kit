# 架构设计

> 最后更新：2026-06-28

## 系统概览

```
┌─────────────────────────────────────────────────────────────┐
│ Frontend                                                    │
│ Vue3 + Vite + Pinia + Tailwind + Element Plus Message       │
└─────────────────────────────┬───────────────────────────────┘
                              │ HTTP / JWT
┌─────────────────────────────▼───────────────────────────────┐
│ Backend API                                                  │
│ Django + DRF + SimpleJWT                                     │
│ core / users / notebooks / documents / chat                  │
└─────────────────────────────┬───────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐     ┌────────────────┐    ┌────────────────┐
│ Storage       │     │ Async Tasks     │    │ AI Providers    │
│ MySQL/SQLite  │     │ Celery + Redis  │    │ DeepSeek/Tavily  │
│ local media   │     │ parsing pipeline│    │ OCR/Vision       │
└───────────────┘     └────────────────┘    └────────────────┘
```

当前实现是轻量单体后端 + 独立前端。向量数据库、对象存储、Electron 桌面端和完整生产容器编排属于后续规划，不是当前运行依赖。

## 后端模块

| 模块 | 路径 | 职责 |
|------|------|------|
| core | `backend/apps/core/` | 健康检查、统一响应渲染、异常处理 |
| users | `backend/apps/users/` | 注册、登录、刷新、退出、当前用户 |
| notebooks | `backend/apps/notebooks/` | Notebook CRUD、收藏、搜索、分页 |
| documents | `backend/apps/documents/` | 上传、校验、解析、分块、图片资源、OCR、视觉描述、重新解析 |
| chat | `backend/apps/chat/` | 会话、消息、RAG、联网搜索、LLM 调用、引用来源 |

## 前端模块

| 模块 | 路径 | 职责 |
|------|------|------|
| app | `frontend/src/app/` | 应用入口、路由与认证后壳层 |
| config | `frontend/src/config/` | 品牌、导航与 Starter module manifest |
| features | `frontend/src/features/` | marketing、auth、dashboard、knowledge、chat、admin、billing |
| components | `frontend/src/components/` | 跨 feature 复用的品牌、导航、文档、聊天与基础 UI |

| stores | `frontend/src/stores/` | Pinia 状态管理 |

## 文档解析链路

```
上传文件
  → 保存到 backend/media
  → 创建 Document 记录
  → parse_document_task
  → parse_file_blocks(TXT/MD/PDF/DOCX)
  → chunk_blocks
  → extract_assets / replace_document_assets
  → build_ocr_blocks
  → build_vision_blocks
  → 写入 DocumentChunk
  → 更新状态与诊断字段
```

关键点：

- Markdown、DOCX、PDF 会尽量保留标题、表格、代码、页码等结构元数据。
- 图片会登记为 `DocumentAsset`，OCR 和视觉描述结果会生成额外检索片段。
- OCR/视觉失败不会让整个文档失败；错误会通过文档卡片诊断字段展示。
- `POST /api/v1/documents/{id}/reparse/` 可让旧文档重新进入解析链路。

## Chat / RAG 链路

```
用户问题
  → ChatService
  → search_mode(local/web/hybrid)
  → 本地结构感知检索 + Tavily 可选联网搜索
  → 证据重排、去重、多样性选择
  → 构建 Prompt
  → DeepSeek Chat Completions
  → 保存 user/assistant message
  → 返回 citations
```

当前本地检索能力：

- 关键词召回。
- 图片、表格、代码、标题等问题意图识别。
- 结构类型加权、重排和去重。
- citation metadata 暴露 `retrieval_score`、`retrieval_reason`、`source_type`。
- 图片视觉描述 citation 暴露 `vision_provider`、`vision_model`。

## 数据存储

## 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + Vite + Pinia | 单页应用 |
| UI | Tailwind CSS + Element Plus Message | 自定义基础组件 + 消息提示服务 |
| 后端 | Django 5 + DRF | REST API 和业务逻辑 |
| 认证 | SimpleJWT | JWT 轮换和黑名单 |
| 异步任务 | Celery + Redis | 文档解析任务 |
| 数据库 | MySQL / SQLite | 生产推荐 MySQL，本地可 SQLite |
| 文档解析 | pypdf + python-docx + 自定义 Markdown/TXT 解析 | 结构化分块 |
| OCR | Tesseract | 可选图片文字识别 |
| 视觉模型 | OpenAI-compatible / Zhipu | 可选图片语义描述 |
| LLM | DeepSeek OpenAI-compatible API | 回答生成 |
| Web Search | Tavily | 联网搜索 |

## 当前边界

- 后端仍是轻量单体，模块边界按 Django app 划分。
- RAG 仍基于数据库检索和重排，未接入 embedding/vector store。
- 自托管需要手动配置应用进程和 Nginx；仓库暂未提供生产 Dockerfile。
- 桌面端、移动端、多用户协作和向量数据库属于后续阶段。
