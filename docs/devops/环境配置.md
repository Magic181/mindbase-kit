# 环境配置

> 最后更新：2026-06-27

项目根目录使用单一 `.env` 文件，模板见 `.env.example`。后端启动时从仓库根目录读取 `.env`，前端通过 Vite 代理访问后端 API。

## 一、基础环境

| 工具 | 推荐版本 | 用途 |
|------|----------|------|
| Python | 3.12+ | Django、Celery、文档解析 |
| Node.js | 22 LTS | Vue3 + Vite |
| pnpm | 10+ | 前端依赖管理 |
| Docker Desktop | 最新稳定版 | 本地 MySQL/Redis |
| Tesseract | 5+ | 可选 OCR |

## 二、快速配置

```powershell
Copy-Item .env.example .env
```

最小本地配置：

```env
USE_SQLITE=true
CELERY_TASK_ALWAYS_EAGER=true
DEEPSEEK_API_KEY=
TAVILY_API_KEY=
```

使用 MySQL/Redis 时先启动开发依赖：

```powershell
docker compose -f docker-compose.dev.yml up -d
```

## 三、核心变量

### Django

| 变量 | 默认值 | 说明 |
|------|--------|------|
| DJANGO_SECRET_KEY | dev-insecure-key | 生产环境必须替换 |
| DJANGO_DEBUG | true | 生产环境设置为 false |
| DJANGO_ALLOWED_HOSTS | localhost,127.0.0.1 | 生产域名白名单，逗号分隔 |
| DJANGO_CORS_ALLOWED_ORIGINS | http://localhost:5173,http://127.0.0.1:5173 | 允许访问 API 的前端来源，逗号分隔 |

### Database

| 变量 | 默认值 | 说明 |
|------|--------|------|
| USE_SQLITE | 空 | `true` 时使用 `backend/db.sqlite3` |
| MYSQL_HOST | 空 | 非空且 `USE_SQLITE=false` 时启用 MySQL |
| MYSQL_PORT | 3306 | MySQL 端口 |
| MYSQL_DATABASE | ai_notebook | 数据库名 |
| MYSQL_USER | ai_notebook | 数据库用户 |
| MYSQL_PASSWORD | dev-password | 数据库密码 |

### Redis / Celery

| 变量 | 默认值 | 说明 |
|------|--------|------|
| REDIS_HOST | localhost | Redis 主机，主要供组合配置参考 |
| REDIS_PORT | 6379 | Redis 端口 |
| CELERY_BROKER_URL | redis://localhost:6379/0 | Celery broker |
| CELERY_RESULT_BACKEND | 同 broker | Celery result backend |
| CELERY_TASK_ALWAYS_EAGER | false | `true` 时解析任务同步执行 |
| MAX_UPLOAD_SIZE_MB | 20 | 单文件上传大小限制 |

### JWT

| 变量 | 默认值 | 说明 |
|------|--------|------|
| JWT_ACCESS_TOKEN_MINUTES | 5 | Access Token 有效期 |
| JWT_REFRESH_TOKEN_DAYS | 7 | Refresh Token 有效期 |

## 四、AI 与搜索

### DeepSeek

DeepSeek 使用 OpenAI-compatible Chat Completions 接口。

| 变量 | 默认值 | 说明 |
|------|--------|------|
| DEEPSEEK_BASE_URL | https://api.deepseek.com/v1 | API 地址 |
| DEEPSEEK_MODEL | deepseek-v4-flash | Chat 模型 |
| DEEPSEEK_API_KEY | 空 | 必填，未填会返回友好降级 |
| DEEPSEEK_TIMEOUT_SECONDS | 60 | 请求超时 |
| DEEPSEEK_MAX_RETRIES | 2 | 可恢复错误重试次数 |
| DEEPSEEK_RETRY_BACKOFF_SECONDS | 0.2 | 重试退避秒数 |

### Tavily

| 变量 | 默认值 | 说明 |
|------|--------|------|
| TAVILY_API_KEY | 空 | 联网搜索 Key，未配置时 web/hybrid 会降级 |
| TAVILY_MAX_RESULTS | 5 | 返回结果数 |
| TAVILY_SEARCH_DEPTH | basic | 搜索深度 |
| TAVILY_TIMEOUT_SECONDS | 20 | 请求超时 |
| TAVILY_MAX_RETRIES | 1 | 重试次数 |
| TAVILY_RETRY_BACKOFF_SECONDS | 0.2 | 重试退避秒数 |

## 五、OCR 与视觉理解

### OCR

| 变量 | 默认值 | 说明 |
|------|--------|------|
| OCR_ENABLED | true | 是否启用图片 OCR |
| OCR_TESSERACT_CMD | tesseract | Tesseract 命令或绝对路径 |
| OCR_LANG | chi_sim+eng | OCR 语言 |
| OCR_TIMEOUT_SECONDS | 30 | 单张图片 OCR 超时 |

OCR 失败不会中断文档解析；文档卡片会显示 OCR 成功、跳过或失败数量。

### Vision

| 变量 | 默认值 | 说明 |
|------|--------|------|
| VISION_ENABLED | false | 是否启用图片视觉描述 |
| VISION_PROVIDER | openai_compatible | `openai_compatible` 或 `zhipu` |
| VISION_API_KEY | 空 | 视觉模型 API Key |
| VISION_BASE_URL | https://api.openai.com/v1 | API 地址 |
| VISION_MODEL | gpt-4o-mini | 视觉模型 |
| VISION_TIMEOUT_SECONDS | 60 | 请求超时 |
| VISION_RETRY_ATTEMPTS | 2 | 每个模型的尝试次数 |
| VISION_RETRY_BACKOFF_SECONDS | 1 | 重试退避秒数 |
| VISION_FALLBACK_MODELS | 空 | 备用模型，逗号分隔 |
| VISION_THINKING | 空 | 智谱可设置 `enabled` 或 `disabled` |
| VISION_PROMPT | 内置中文提示词 | 自定义图片视觉描述提示词 |

智谱 GLM 视觉示例：

```env
VISION_ENABLED=true
VISION_PROVIDER=zhipu
VISION_BASE_URL=https://open.bigmodel.cn/api/paas/v4
VISION_MODEL=glm-4.6v-flashx
VISION_API_KEY=
VISION_THINKING=disabled
```

## 六、RAG

| 变量 | 默认值 | 说明 |
|------|--------|------|
| RAG_TOP_K | 5 | 默认引用片段数量 |
| RAG_MAX_CONTEXT_CHARS | 8000 | 注入 LLM 的最大上下文字符数 |

当前检索使用数据库关键词召回、结构类型意图增强、重排和去重；向量数据库属于后续升级项。

## 七、推荐组合

### 最小本地体验

```env
USE_SQLITE=true
CELERY_TASK_ALWAYS_EAGER=true
OCR_ENABLED=false
VISION_ENABLED=false
```

### 本地完整解析

```env
USE_SQLITE=true
CELERY_TASK_ALWAYS_EAGER=true
OCR_ENABLED=true
VISION_ENABLED=true
VISION_PROVIDER=zhipu
VISION_MODEL=glm-4.6v-flashx
```

### 接近生产

```env
DJANGO_DEBUG=false
DJANGO_ALLOWED_HOSTS=your-domain.com
DJANGO_CORS_ALLOWED_ORIGINS=https://your-domain.com
USE_SQLITE=false
MYSQL_HOST=127.0.0.1
CELERY_TASK_ALWAYS_EAGER=false
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
```
