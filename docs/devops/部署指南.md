# 部署指南

> 最后更新：2026-06-27

本指南描述当前仓库真实可运行的部署方式。当前代码包含 Vue3 前端、Django API、Celery 任务、MySQL/SQLite、Redis、DeepSeek/Tavily、OCR 和可选视觉模型；尚未包含完整生产 Dockerfile、Milvus、MinIO 或 Electron 桌面打包。

## 部署模式

| 模式 | 适合场景 | 当前状态 |
|------|----------|----------|
| 本地开发 | 日常开发、调试、批次验证 | 已支持 |
| 本地一体化体验 | SQLite + eager Celery，最少依赖启动 | 已支持 |
| 基础自托管 | 服务器上手动运行前端静态资源、Django、Celery、MySQL、Redis | 可按本指南配置 |
| 完整容器化 / 桌面端 | Dockerfile、Nginx 镜像、Electron 打包 | 后续规划 |

## 目录结构

```
AI-Notebook/
├── backend/                 # Django API、Celery、文档解析、RAG
│   ├── apps/
│   │   ├── core/            # 健康检查、统一响应、异常处理
│   │   ├── users/           # 认证
│   │   ├── notebooks/       # Notebook 管理
│   │   ├── documents/       # 上传、解析、OCR、视觉描述、重新解析
│   │   └── chat/            # 会话、RAG、联网搜索、LLM 调用
│   └── config/              # Django/Celery 配置
├── frontend/                # Vue3 + Vite 前端
├── docs/                    # 产品、工程、AI、设计、运维文档
├── docker-compose.dev.yml   # 开发依赖：MySQL + Redis
└── .env.example             # 环境变量模板
```

## 一、本地开发

### 1. 准备环境

| 工具 | 推荐版本 |
|------|----------|
| Python | 3.12+ |
| Node.js | 22 LTS |
| pnpm | 10+ |
| Docker Desktop | 最新稳定版 |
| Git | 最新稳定版 |

### 2. 配置环境变量

```powershell
Copy-Item .env.example .env
```

快速体验建议先使用 SQLite 和同步 Celery：

```env
USE_SQLITE=true
CELERY_TASK_ALWAYS_EAGER=true
DEEPSEEK_API_KEY=
TAVILY_API_KEY=
```

如果需要 MySQL/Redis：

```powershell
docker compose -f docker-compose.dev.yml up -d
```

`docker-compose.dev.yml` 当前只提供：

- MySQL: `localhost:3306`
- Redis: `localhost:6379`

### 3. 启动后端

```powershell
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

后端默认地址：`http://localhost:8000`

### 4. 启动前端

```powershell
cd frontend
pnpm install
pnpm dev
```

前端默认地址：`http://localhost:5173`

### 5. Celery Worker

如果 `.env` 中 `CELERY_TASK_ALWAYS_EAGER=true`，解析任务会在请求进程内同步执行，不需要额外 worker。

如果改为异步队列：

```powershell
cd backend
.\venv\Scripts\activate
celery -A config worker -l info
```

## 二、OCR 与视觉模型

### OCR

OCR 使用本机 Tesseract。开启后，文档解析会提取图片 OCR 文本并写入检索片段。

```env
OCR_ENABLED=true
OCR_TESSERACT_CMD=tesseract
OCR_LANG=chi_sim+eng
OCR_TIMEOUT_SECONDS=30
```

Windows 需要安装 Tesseract，并确保 `OCR_TESSERACT_CMD` 能被后端进程访问。

### 视觉理解

视觉理解用于为图片生成中文描述，适合流程图、架构图、截图和表格图片。

```env
VISION_ENABLED=true
VISION_PROVIDER=zhipu
VISION_BASE_URL=https://open.bigmodel.cn/api/paas/v4
VISION_MODEL=glm-4.6v-flash
VISION_API_KEY=
VISION_TIMEOUT_SECONDS=60
VISION_RETRY_ATTEMPTS=2
VISION_RETRY_BACKOFF_SECONDS=1
VISION_FALLBACK_MODELS=
VISION_THINKING=disabled
```

说明：

- `VISION_PROVIDER=zhipu` 会按智谱 GLM 视觉接口发送图片。
- `VISION_PROVIDER=openai_compatible` 用于 OpenAI 风格多模态接口。
- `VISION_FALLBACK_MODELS` 可填逗号分隔备用模型；主模型繁忙、限流或 5xx 时会自动尝试。
- 已解析旧文档需要在前端点击“重新解析”才会生成新的 OCR/视觉片段。

## 三、基础自托管

当前仓库未提供生产 Dockerfile。基础自托管可按下面方式部署：

1. 服务器安装 Python、Node.js、pnpm、MySQL、Redis、Nginx。
2. 拉取代码并创建 `.env`。
3. 使用 MySQL，关闭 Debug：

```env
DJANGO_DEBUG=false
DJANGO_ALLOWED_HOSTS=your-domain.com
DJANGO_CORS_ALLOWED_ORIGINS=https://your-domain.com
USE_SQLITE=false
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_DATABASE=ai_notebook
MYSQL_USER=ai_notebook
MYSQL_PASSWORD=your-secure-password
CELERY_TASK_ALWAYS_EAGER=false
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
```

4. 构建前端：

```bash
cd frontend
pnpm install
pnpm build
```

5. 初始化后端：

```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic
```

`collectstatic` 会收集到后端 `staticfiles/` 目录，生产环境可由 Nginx 托管该目录。

6. 用 systemd/supervisor 管理：

- `python manage.py runserver 127.0.0.1:8000` 仅适合临时验证。
- 生产建议接入 WSGI/ASGI 服务，并用 Nginx 反向代理 `/api/`。
- Celery worker 独立进程运行：`celery -A config worker -l info`。

## 五、常见问题

### 上传后一直解析中

- 如果使用异步 Celery，确认 Redis 和 worker 正常运行。
- 本地快速体验可设置 `CELERY_TASK_ALWAYS_EAGER=true`。

### OCR 没有结果

- 确认安装 Tesseract。
- 确认 `OCR_ENABLED=true`。
- 中文图片需要 `OCR_LANG=chi_sim+eng` 且本机安装对应语言包。

### 视觉识别失败

- `VISION_API_KEY not configured`: 未配置视觉模型 API Key。
- `鉴权失败`: 检查 Key、模型权限、账户额度。
- `模型当前访问量过大` 或 `1305`: 服务繁忙，稍后重新解析，或配置 `VISION_FALLBACK_MODELS`。
- `请求超时`: 调大 `VISION_TIMEOUT_SECONDS` 或降低并发。

### 问答引用不符合预期

- 确认文档状态为 `completed`。
- 图片、表格、代码类问题依赖结构化片段，旧文档需要重新解析。
- 混合搜索需要配置 `TAVILY_API_KEY`。
