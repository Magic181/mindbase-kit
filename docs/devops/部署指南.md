# 部署指南

> 最后更新：2026-07-09

本指南覆盖 MindBase Kit 的本地开发、Docker 一键部署、生产配置和排障。当前仓库已包含前端、后端、worker、MySQL、Redis 和 Nginx 反向代理的完整 Compose 编排。

## 部署模式

| 模式 | 适合场景 | 命令 |
|------|----------|------|
| 本地开发 | 日常开发、调试前后端 | 前后端分别启动 |
| 开发依赖服务 | 本机跑代码，Docker 跑 MySQL/Redis | `docker compose -f docker-compose.dev.yml up -d` |
| 一键部署 | 本机或服务器验证完整栈 | `docker compose up -d --build` |

## 一、Docker 一键部署

### 1. 准备环境

- Docker Desktop 或 Docker Engine
- Git

### 2. 创建环境变量

```bash
cp .env.example .env
```

生产或公网部署前必须修改：

```env
DJANGO_SECRET_KEY=replace-with-a-long-random-secret
DJANGO_DEBUG=false
DJANGO_ALLOWED_HOSTS=your-domain.com,localhost,127.0.0.1
DJANGO_CORS_ALLOWED_ORIGINS=https://your-domain.com,http://localhost:8080
MYSQL_PASSWORD=replace-with-a-strong-password
DOCKER_MYSQL_ROOT_PASSWORD=replace-with-a-strong-root-password
DOCKER_MYSQL_PASSWORD=replace-with-a-strong-app-password
DEEPSEEK_API_KEY=
```

### 3. 启动

```bash
docker compose up -d --build
```

访问：

- Landing Page: `http://localhost:8080`
- Demo 应用: `http://localhost:8080/app`
- API 健康检查: `http://localhost:8080/api/v1/health/`

### 4. 创建管理员

```bash
docker compose exec backend python manage.py createsuperuser
```

管理后台：

```text
http://localhost:8080/admin/
```

### 5. 查看日志

```bash
docker compose logs -f backend
docker compose logs -f worker
docker compose logs -f frontend
```

## 二、Compose 服务说明

| 服务 | 镜像来源 | 说明 |
|------|----------|------|
| `frontend` | `frontend/Dockerfile` | 构建 Vite 静态资源，Nginx 托管并代理 API |
| `backend` | `backend/Dockerfile` | Django + Gunicorn，启动时自动 migrate 和 collectstatic |
| `worker` | `backend/Dockerfile` | Celery worker，处理文档解析等异步任务 |
| `mysql` | `mysql:8.0` | 生产式默认数据库 |
| `redis` | `redis:7-alpine` | Celery broker/result backend |

持久化卷：

- `mysql_data`: MySQL 数据。
- `redis_data`: Redis 数据。
- `backend_media`: 上传文件和解析资产。
- `backend_static`: Django staticfiles。

## 三、本地开发

### 1. 后端

```powershell
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

后端默认地址：`http://localhost:8000`

### 2. 前端

```powershell
cd frontend
pnpm install
pnpm dev
```

前端默认地址：`http://localhost:5173`

### 3. 仅启动开发依赖

```powershell
docker compose -f docker-compose.dev.yml up -d
```

提供：

- MySQL: `localhost:3306`
- Redis: `localhost:6379`

## 四、OCR 与视觉模型

Docker 后端镜像已安装英文与简体中文 Tesseract 语言包。Windows 本地开发仍需自行安装 Tesseract。

OCR：

```env
OCR_ENABLED=true
OCR_TESSERACT_CMD=tesseract
OCR_LANG=chi_sim+eng
OCR_TIMEOUT_SECONDS=30
```

视觉理解：

```env
VISION_ENABLED=true
VISION_PROVIDER=openai_compatible
VISION_BASE_URL=https://api.openai.com/v1
VISION_MODEL=gpt-4o-mini
VISION_API_KEY=
```

## 六、常见问题

### 上传后一直解析中

- 确认 `worker` 正常运行：`docker compose logs -f worker`。
- 确认 Redis 健康：`docker compose ps redis`。
- 本地快速体验可设置 `CELERY_TASK_ALWAYS_EAGER=true`，但 Docker 一键部署默认使用异步 worker。

### API 401 后跳转登录

- 前端会自动刷新 access token。
- refresh token 失效后会跳转 `/login`。
- 确认浏览器没有旧 token，必要时清空 localStorage。

### OCR 没有结果

- Docker 模式确认 `OCR_ENABLED=true`。
- 本地 Windows 确认 `OCR_TESSERACT_CMD` 能被后端进程访问。
- 中文图片需要 `OCR_LANG=chi_sim+eng` 且安装对应语言包。

### 视觉识别失败

- `VISION_API_KEY not configured`: 未配置视觉模型 API Key。
- `鉴权失败`: 检查 Key、模型权限、账户额度。
- `请求超时`: 调大 `VISION_TIMEOUT_SECONDS` 或降低并发。

### 问答引用不符合预期

- 确认文档状态为 `completed`。
- 图片、表格、代码类问题依赖结构化片段，旧文档需要重新解析。
- 混合搜索需要配置 `TAVILY_API_KEY`。

## 七、服务器发布提示

1. 将 `8080:80` 改成 `80:80`，或在外层 Nginx/Caddy 反代到 `frontend:80`。
2. 设置真实域名的 `DJANGO_ALLOWED_HOSTS` 和 `DJANGO_CORS_ALLOWED_ORIGINS`。
3. 使用强密码替换 MySQL 默认密码。Docker 一键部署使用 `DOCKER_MYSQL_*`，本地开发连接外部 MySQL 使用 `MYSQL_*`。
4. 对 `backend_media`、`mysql_data` 做备份。
5. 公网环境关闭 `DJANGO_DEBUG`。
