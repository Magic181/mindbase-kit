# 部署指南

## 部署模式概览

AI Notebook 支持三种部署模式，覆盖开发、自托管和桌面端全场景。

```
AI Notebook 部署
├── Development（开发环境）
│   └── Windows / macOS / Linux 本地开发
├── Self-host（自托管部署）
│   └── Docker Compose 一键部署
└── Desktop（桌面版）
    └── Electron 打包，双击即用
```

---

## 一、Development（开发环境）

### 环境要求

#### 操作系统

- **Windows 11**（推荐）
- macOS（支持）
- Ubuntu 22.04+（支持）

#### 开发工具

| 工具 | 版本 | 说明 |
|------|------|------|
| Python | 3.12+ | 性能更优，长期支持 |
| Node.js | 22 LTS | Vue3 + Vite 最佳兼容 |
| pnpm | 10+ | 快速、节省磁盘 |
| Docker Desktop | 最新 | 容器化服务 |
| Git | 最新 | 版本控制 |
| VS Code / Cursor | 最新 | 开发IDE |

### 快速开始

#### 1. 克隆项目

```bash
git clone https://github.com/your-org/ai-notebook.git
cd ai-notebook
```

#### 2. 安装前端依赖

```bash
cd frontend
pnpm install
```

#### 3. 安装后端依赖

```bash
cd backend
python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

pip install -r requirements.txt
```

#### 4. 启动基础设施

```bash
docker compose -f docker-compose.dev.yml up -d
```

启动服务：
- MySQL
- Redis
- Milvus（或 ChromaDB）
- MinIO

#### 5. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 填入配置
```

#### 6. 初始化数据库

```bash
cd backend
python manage.py migrate
python manage.py createsuperuser
```

#### 7. 启动开发服务

**终端1 - 前端：**
```bash
cd frontend
pnpm dev
```

**终端2 - 后端：**
```bash
cd backend
python manage.py runserver
```

**终端3 - Celery Worker：**
```bash
cd backend
celery -A config worker -l info
```

#### 8. 访问应用

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:5173 |
| 后端API | http://localhost:8000/api/v1/ |
| 管理后台 | http://localhost:8000/admin/ |
| MinIO控制台 | http://localhost:9001 |

## 二、Self-host（自托管部署）

适用于个人服务器、团队内部部署。

### 环境要求

#### 服务器配置

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 100GB+ SSD |
| 网络 | 10Mbps | 100Mbps+ |

#### 软件环境

- **操作系统**：Ubuntu 24.04 LTS（推荐）
- Docker + Docker Compose
- Nginx
- SSL证书（Let's Encrypt）

### 部署步骤

#### 1. 准备服务器

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# 安装 Docker Compose
sudo apt install docker-compose-plugin -y
```

#### 2. 克隆项目

```bash
git clone https://github.com/your-org/ai-notebook.git
cd ai-notebook
```

#### 3. 配置环境变量

```bash
cp .env.example .env
vim .env
```

关键配置：

```env
# Django
DJANGO_SECRET_KEY=your-secret-key
DJANGO_DEBUG=false
DJANGO_ALLOWED_HOSTS=your-domain.com

# Database
MYSQL_ROOT_PASSWORD=your-root-password
MYSQL_DATABASE=ai_notebook
MYSQL_USER=ai_notebook
MYSQL_PASSWORD=your-password

# AI（按需配置）
OPENAI_API_KEY=sk-xxx
```

#### 4. 一键启动

```bash
docker compose up -d
```

#### 5. 初始化

```bash
docker compose exec backend python manage.py migrate
docker compose exec backend python manage.py createsuperuser
```

#### 6. 配置 Nginx

```nginx
# /etc/nginx/sites-available/ai-notebook
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # 前端
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        client_max_body_size 2G;
    }

    # WebSocket（聊天流式响应）
    location /ws {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/ai-notebook /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 7. 配置 SSL

```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com
```

### 数据备份

```bash
# 数据库备份
docker compose exec mysql mysqldump -u root -p ai_notebook > backup_$(date +%Y%m%d).sql

# 恢复
docker compose exec mysql mysql -u root -p ai_notebook < backup.sql

# 文件备份（MinIO数据卷）
docker run --rm -v ai-notebook_minio_data:/data -v $(pwd):/backup alpine tar czf /backup/minio_backup.tar.gz /data
```

## 三、Desktop（桌面版）

基于 Electron 打包，最终用户双击即可使用。

### 技术方案

| 组件 | 技术 | 说明 |
|------|------|------|
| 框架 | Electron | 跨平台桌面应用 |
| 前端 | Vue3 | 复用Web版代码 |
| 后端 | 内嵌Python | 打包进应用 |
| 数据库 | SQLite | 替代MySQL，轻量化 |
| 向量库 | ChromaDB | 替代Milvus，嵌入式 |

### 打包流程

```
┌─────────────────────────────────────────────────────────┐
│                   Electron 打包                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │                 AI Notebook                      │   │
│  │                  Electron                        │   │
│  ├─────────────────────────────────────────────────┤   │
│  │  ┌─────────────┐  ┌─────────────────────────┐  │   │
│  │  │   Vue3 UI   │  │    Embedded Backend     │  │   │
│  │  │   (Web)     │  │    Python + SQLite      │  │   │
│  │  └─────────────┘  └─────────────────────────┘  │   │
│  │  ┌─────────────────────────────────────────┐   │   │
│  │  │              AI Engine                   │   │   │
│  │  │    ChromaDB + Ollama (本地) 或 API       │   │   │
│  │  └─────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────┘   │
│                         │                               │
│                         ▼                               │
│  ┌─────────────────────────────────────────────────┐   │
│  │                  打包产物                        │   │
│  │  ├── Windows.exe                                │   │
│  │  ├── macOS.app                                  │   │
│  │  └── Linux.AppImage                             │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 本地运行（开发模式）

```bash
# 安装依赖
cd desktop
pnpm install

# 开发模式运行
pnpm dev

# 打包
pnpm build
```

### 打包配置

```json
// desktop/package.json
{
  "build": {
    "appId": "com.ainotebook.desktop",
    "productName": "AI Notebook",
    "directories": {
      "output": "release"
    },
    "files": [
      "dist/**/*",
      "backend/**/*"
    ],
    "win": {
      "target": "nsis",
      "icon": "build/icon.ico"
    },
    "mac": {
      "target": "dmg",
      "icon": "build/icon.icns"
    },
    "linux": {
      "target": "AppImage",
      "icon": "build/icon.png"
    }
  }
}
```

## 六、常见问题

### 开发环境

**Q: Docker 服务启动失败？**
```bash
# 检查端口占用
netstat -tulpn | grep :3306
# 重启 Docker Desktop
```

**Q: pnpm install 报错？**
```bash
# 清除缓存
pnpm store prune
rm -rf node_modules
pnpm install
```

### 生产环境

**Q: Nginx 502 Bad Gateway？**
```bash
# 检查后端服务
docker compose ps
docker compose logs backend
```

**Q: SSL证书过期？**
```bash
sudo certbot renew
sudo systemctl reload nginx
```

### 桌面版

**Q: 打包后无法启动？**
- 检查 Python 环境是否正确嵌入
- 检查 SQLite 数据库路径权限
